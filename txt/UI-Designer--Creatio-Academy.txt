**Application UI** 
 provides a way for users to interact with your Creatio application. It is a set of pages and boxes that have various controls. Users can interact with controls, for example, using a keyboard and mouse. You can create and set up custom UIs of any structure and place elements anywhere on the canvas. For example, set up tabs that display analytics in the section list.
 



 The
 **UI Designer** 
 is a WYSIWYG editor that lets you create and set up mobile responsive appearance of apps. The pages are set up in a new UI framework, which lets you modify the element appearance, manage data sources, and place page components arbitrarily.
 



 You can use the UI Designer to customize lists and pages of any structure. Use no-code tools to work in the UI Designer. UI creation involves structuring the page with tabs, areas, columns. Depending on the information you want to display on the page, you can also set up charts, add fields, data sources, buttons, and other components.

 UI Designer elements
----------------------



 Create UIs in the UI Designer using page elements: fields, charts, buttons, text, and their various combinations.
 



 View the elements of the UI Designer in the table below:
 





| 
 Element types
  | 
 Description
  |
| --- | --- |
| [Layout elements](#title-2200-1)  | 
 Elements that structure the page: expansion panels, tabs, areas, columns.
  |
| [Data](#title-2200-28)  | 
 Elements that display the fields of the current object. The fields of the current object cannot be deleted.
  |
| [Inputs](#title-2200-9)  | 
 Elements that display data and let you enter data. The following field types are available: text, number, dropdown, date/time, checkbox.
  |
| [Components](#title-2200-16)  | 
 Elements that execute target page actions and solve a number of other problems: button, label.
  |
| [Charts](#title-2200-19)  | 
 Elements that visualize statistics. The following chart types are available: spline, line, bar, doughnut, area, column, scatter, pipeline, metric, sales pipeline, full pipeline.
  |

 Column spacing

 Number of columns

2. Select the distance between columns in the
 
 Column spacing

 Data elements
---------------


### 
 Manage object columns



 Use
 
 Data
 
 layout element to display data on the page or manage input fields. You can create new attributes in the data source or use existing fields. The area is split in two parts: new columns and existing columns.
 



 To add existing fields to the page, drag them from the Data area to the relevant part of the canvas.
 



 Creatio specifies the data source automatically.
 


### 
 Fields



 Use the
 
 Fields
 
 layout element to display data and enable users to enter data.
 



 Creatio adds a new column to the data source after you add a new field to the app page and save the changes.
 



 You can add the following fields types to the page in the UI Designer (Fig. 11):
 


* **String** 
 . Enables users to enter letters, digits, and other characters.
* **Number** 
 . Enables users to enter integers or fractions.
* **Dropdown** 
 . Enables users to select values from a set list of options stored in a lookup or other data source.
* **Date/Time** 
 . Enables users to select date and/or time, depending on the selected format.
* **Boolean** 
 . Enables users to select either of the two values: “True” or “False.” “True” value is a selected checkbox.




 Fig. 11 Adding various field types in the UI Designer
 

![add_fields_designer.gif](/docs/sites/en/files/images/NoCode_Customization/designer_interfaces/add_fields_designer.gif)



 The UI designer supports various properties that provide flexible field setup options. For example, requirement flag, read-only mode, multi-line mode, etc. Set these properties in the field setup area (Fig. 12):
 


1. Select the
 
 Read-only
 
 checkbox in the
 
 Editability
 
 group to make the field non-editable.
2. Enter the title the users will see on the app page in the
 
 Title on page
 
 field in the
 
 Appearance
 
 group.
3. Specify the data source from which to populate the field in the
 
 Data source
 
 field in the
 
 Advanced
 
 group. Once you do this, Creatio will bring up the
 
 General
 
 group and populate the following fields automatically:
	1. Title
	 
	 . Displays the field name the users will see. For example, “Employer.” The field is required.
	2. Code
	 
	 . Displays the unique name Creatio will use to create an object column. For example, the code for the
	 
	 Employer
	 
	 field can be “UsrEmployer.” The code can only contain alphanumeric characters. The field becomes locked after you save the changes or switch to editing the section page.
	3. Format
	 
	 . Sets the display format of text, numbers, dates, and time values. The field is required.
	4. Description
	 
	 . Displays additional information about the field.
	5. Required
	 
	 . Flags the field as required on the app page. The field is required.
	6. Lookup
	 
	 . Specifies the object from which the app requests available field options. The field is required.
	7. Copy this value when copying records
	 
	 . Enables copying the field data when copying the record.

 Components
------------


### 
 Button



 Use the
 
 Button
 
 layout element to execute a page action. For example, save the changes or run a process.
 



 Set display and operation parameters of a button in the setup area (Fig. 13):
 


1. Enter the button caption in the
 
 Title
 
 field.
2. Select the button fill color in the
 
 Style
 
 field. This lets you color code the intended function of a button.
3. Specify the action to execute when the user clicks the button in the
 
 Action
 
 field: save the record, close the page, cancel the changes, or run a process.
4. Select an existing process in the
 
 Which process to run?
 
 field or set up a new process. The field appears if you specify “Run process” in the
 
 Action
 
 field.
 



 If the selected process uses parameters,
 
 How to run the process?
 
 field appears. To run the process for the selected page, specify the process parameter in which to pass the ID of the current record in the additional field that appears.





 Note.
 
 Track the process flow in the
 
 Process log
 
 section.
 





 Fig. 13 Button setup area
 

![add_button_settings_0.png](/docs/sites/en/files/images/NoCode_Customization/designer_interfaces/add_button_settings_0.png)


### 
 Label



 Use the
 
 Label
 
 layout element to add arbitrary text to the page, as well as create individual headings for element groups (Fig. 14).
 




 Fig. 14 Adding a label in the UI Designer
 

![add_title_designer.gif](/docs/sites/en/files/images/NoCode_Customization/designer_interfaces/add_title_designer.gif)



 Edit label text and alignment, font style and format, as well as select text and background color in the element setup area (Fig. 15):
 


1. Enter the label text to display on the app page in the
 
 Text
 
 field. The field is required.
2. Select the label style to display on the app page in the
 
 Style
 
 field. For example, “H1.” The field is required.
3. Select the font format in the
 
 Format
 
 field. For example, “Semibold.” The field is required.
4. Select the text alignment option in the
 
 Align
 
 field: left, center, right, justify.
5. Select the text color in the
 
 Text color
 
 field.
6. Select the label background color in the
 
 Background color
 
 field.
